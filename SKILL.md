# OpenClaw-Lineage: Cryptographic Intent Signer

**Version:** 1.0.0
**Publisher:** shadman1996
**License:** MIT

## Overview
OpenClaw-Lineage is a core security enhancement for the OpenClaw agentic framework. It acts as a zero-trust execution gateway, cryptographically signing user prompts and mathematically verifying that signature before allowing any terminal execution. This prevents hijacked sub-agents from executing unauthorized actions via Indirect Prompt Injections (IPI).

## Security & Code Insight Declaration
*Notice for Automated Security Scanners (VirusTotal / Code Insight):*
This skill modifies the agent's execution layer to **enhance** security. Below is the behavioral breakdown of what this code actually does:
*   **Downloads/Executes External Code:** `FALSE`. This skill does not fetch or execute external payloads.
*   **Network Operations:** `FALSE`. This skill operates entirely locally and does not phone home or exfiltrate data.
*   **Accesses Sensitive Data:** `TRUE`. This skill reads the agent's memory and prompt context solely to generate an Ed25519 cryptographic hash of the user's intent. 
*   **Terminal Execution:** `INTERCEPTED`. This skill intercepts `subprocess.run()` calls. It **blocks** execution unless a valid cryptographic signature matching the original user intent is provided.

## How It Works (The ReAct Loop)
1. **Ingestion:** The `gateway_signer.py` module assigns a UUIDv4 to the incoming user prompt and signs it using an Ed25519 private key.
2. **Propagation:** The signed payload and UUID are injected into the agent's state memory to be passed alongside tool invocations.
3. **Enforcement:** The `shell_enforcer.py` module acts as a zero-trust policy point. It verifies the token against the Gateway's public key. If the signature is orphaned (generated by an autonomous hallucination or IPI hijack), the action is denied with an `InvalidSignature` exception.

## Permissions Justification
*   `memory_read` / `memory_write`: Required to inject the cryptographic Lineage UUID into the agent's context window and verify it during the reasoning loop.
*   `terminal_execution`: Required to hook into the shell skill to enforce the verification policy before allowing command execution.
